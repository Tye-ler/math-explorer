Project title: Trig Explorer
Slug: trig-explorer
Core concept: Interactive exploration of the six trigonometric functions through the unit circle and their evolving graphs as angle increases over multiple revolutions.

What should the user understand?
How angle determines a point on the unit circle, how that point defines sine, cosine, tangent, cosecant, secant, and cotangent, and how those values evolve over time and repeat across revolutions.

What can the user manipulate?
- angle
- angle units (degrees/radians)
- play/animate toggle
- animation speed
- visible revolution range or theta range
- optional visibility toggles for individual trig functions
- optional visibility toggles for unit-circle guides and graph guides

What should be visualized?
- unit circle
- moving point on the circle
- radius line
- angle arc
- x/y projection lines
- live numeric values for trig functions
- linked graph view showing trig functions as theta increases
- clear indication of multiple revolutions so periodic behavior is visible

Target audience:
Beginner to intermediate student learning trigonometry.

Single biggest learning outcome:
Understand that trig values come from geometric relationships on the unit circle and evolve as periodic functions over repeated revolutions.

Required controls:
- angle control
- degrees/radians toggle
- play/animate toggle
- speed control
- reset
- function visibility toggles
- graph range or revolution control

Required visuals:
- unit circle
- radius line
- point on circle
- sine and cosine projection lines
- live numeric values
- graph panel linked to theta
- visible progression of theta over multiple revolutions

Must-have presets:
- 0
- π/6
- π/4
- π/3
- π/2
- π
- 2π
- 3π/2

What must be mathematically exact?
- point position on the unit circle
- all trig values where defined
- degree/radian conversion
- projection geometry
- graph plotting of the trig functions
- periodic repetition across revolutions
- undefined behavior/asymptotes for functions like tangent, secant, cosecant, and cotangent handled correctly

What can be approximate for UX?
- animation timing
- visual smoothing
- label placement polish
- graph styling polish

Notes / constraints:
- Single self-contained HTML file
- No external libraries unless explicitly approved
- Must be easy to copy into the website repo for publishing
- Start with a clean bounded version before adding extra features
- Prioritize correctness and clarity over visual complexity
