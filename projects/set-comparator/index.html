Set expression evaluator
Supported expression language (this page)

The Expression and Operations tabs share one evaluator. Allowed inputs are exactly those described here; anything else should fail with a clear message.

  • Atoms: named sets (letters/identifiers), built-ins ℕ ℤ ℚ ℝ ℂ, empty set ∅, explicit rosters {…}, power set 𝒫(S) only when S resolves to an exact finite extensional set (same rule as the Power Set tab).
  • Operators (binary): ∪ ∩ ∖ △ — parentheses allowed for grouping.
  • Relations: ∈ ∉ ⊆ ⊂ ⊇ ⊃ = ≠ between two set expressions (after normalization).
  • Quantifiers (single form): ∀ x ∈ <domain>, x ∈ <target> and ∃ x ∈ <domain>, x ∈ <target> with a comma at top level separating domain and target.

Not supported: general first-order logic, ∧ ∨ → between formulas, set-builder notation inside expressions, or custom predicates beyond the Set Builder’s stored rules for derived sets.

A ∪ B A ∩ B A ⊆ B A ⊂ B 3 ∈ ℤ 0.5 ∈ ℚ 0 ∈ ℕ 𝒫(A) {1,2,3} ∩ {2,3,4} Commutativity ∀ statement

How to read results
  • Extensional (finite roster): the value is exactly the listed members; counts refer to that roster unless labeled otherwise.
  • Semantic (ℕ, ℤ, …): the set is infinite standard notation; ∈ and relations use built-in rules, not an exhaustive display.
  • Sample / bounded listing (yellow cues): only a finite window is shown — not a proof about the whole derived or infinite set.
  • Symbolic: kept as an unevaluated expression, or the question is not decided in this evaluator.
  • Unsupported / error: outside the documented language or violates domain rules (expand “Supported expression language” above).

Supported rule grammar (derived sets only)
  • Primitives: even, odd
  • Modular: x % m = k, x % m ≠ k (integers m, k; m ≠ 0)
  • Compare x to a constant: x < n, x > n, x <= n, x >= n, x = n, x ≠ n (also == / !=)
  • Combine: and / or / ∧ / ∨ between clauses (no nested arbitrary logic beyond that).

Anything else is rejected at save time with a clear error.

Finite roster mode: the set is exactly the comma-separated elements you list (or ∅ if empty). Switch mode for rule-defined subsets.

Venn diagram (extensional or sample)
What this view can and cannot show
  • Extensional mode (no yellow banner): regions partition the union of members that appear in the listings used for the diagram — a correct partition of those elements, not a claim about unstated elements of infinite sets.
  • Sample projection (yellow banner): overlap is computed on bounded windows (see footnote text). Explore overlap; do not treat it as the exact infinite-set Venn.
  • Canvas cleared with a message: at least one selected set has no finite listing here — use the pairwise notes below and the Expression tab for semantic checks.
Compute Power Set
𝒫(S) is enumerated only when S is exact finite extensional: an explicit roster, or a derived set whose transitive base is such a roster. Infinite standard sets, symbolic expression values, and ℕ/ℤ-derived subsets (infinite, sample-window listings) cannot yield a truthful full power-set listing here.
Why 𝒫 might be unavailable
  • Infinite base: 𝒫(S) would have infinitely many subsets — not enumerated.
  • Sample-only listing: bounded windows from ℕ/ℤ-derived rules do not define a unique finite S for 𝒫.
  • Symbolic 𝒫(…): inner expression did not resolve to an exact finite set — result stays symbolic with an explanation.
